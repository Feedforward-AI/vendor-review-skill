# Executive Summary

Based on the criterion evaluations provided, generate an executive summary for this vendor evaluation.

The summary should:
1. Open with a single, direct statement that answers: "What should we know about this tool, and what trade-offs does it present?"
2. Follow with 2-3 short paragraphs expanding on key strengths and trade-offs
3. Connect observations to the capacity-building framework
4. End with a clear statement of what type of organization this tool would or would not suit

Example opening lines:
- "Vendor X provides strong access to company information, but the opacity of its prompts and model choices raises concerns about long-term capacity building."
- "This tool solves a real problem effectively, but creates significant lock-in through proprietary data formats."
- "A strong choice for organizations prioritizing immediate productivity, with trade-offs for those prioritizing AI capability development."

Determine the overall sentiment:
- "positive": Tool is well-suited with minimal concerns
- "neutral": Meaningful trade-offs that depend on organizational priorities
- "negative": Significant concerns that warrant careful consideration

---

**Scoring vocabulary:** Pass / Partial / Fail / Insufficient

**Key takeaway placement:** The opening sentence IS the key takeaway — make it standalone and quotable.

**Suitability close:** The final sentence must name the type of organization this tool suits (and optionally one it does not). Use language like "best suited for…" or "a strong match for organizations that…".

**Org-context note:** Fold optional org context into the suitability line ONLY; never into the dimension assessments.
